# 시퀀스 다이어그램

본 문서에서는 기존 백엔드 서비스의 주요 API 흐름을 이해하기 위해 대표적인 생성 및 수정 API를 중심으로 시퀀스 다이어그램을 작성하였습니다.

단순 조회 성격의 API는 생성/수정 API와 구조적으로 유사하여 분석 대상에서 제외하였습니다.

## 작성 대상 API
- 게시글 생성 (createPost)
- 게시글 수정 (updatePost)

각 시퀀스 다이어그램은 Controller → Service → Repository → Database 흐름을 기준으로 작성되었으며,
비즈니스 로직(권한 체크 등)은 note를 통해 별도로 표현하였습니다.
