# 시퀀스 다이어그램

본 문서에서는 **기존 백엔드 서비스의 주요 API 흐름을 이해하고, 신규 기능에 대한 설계 의도를 명확히 하기 위해** 대표적인 생성·수정 API와 기능 확장 API를 중심으로 시퀀스 다이어그램을 작성하였습니다.

단순 조회 성격의 API는 생성/수정 API와 구조적으로 유사하여 분석 대상에서 제외하였습니다.

## 작성 대상 API
- 게시글 생성 (createPost)
- 게시글 수정 (updatePost)
- 게시글 좋아요 토글 (toggleLike)
- 인기 게시글 조회 (getPopularPosts)


각 시퀀스 다이어그램은 **Controller → Service → Repository** 흐름을 기준으로 작성하였습니다.
기존 기능 분석을 위한 시퀀스에서는 실제 코드 흐름 이해를 돕기 위해 **Repository → Database** 흐름을 포함하였으며, 신규 기능 설계 시퀀스에서는 구현 상세를 과도하게 노출하지 않기 위해 **Database 계층은 생략** 하고 Repository를 데이터 접근의 추상 계층으로 표현하였습니다.

비즈니스 로직(권한 체크, 예외 발생 등)은 alt 블록 및 메시지를 통해 명시적으로 표현하였습니다.
